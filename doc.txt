ОБЯЗАТЕЛЬНЫЕ тесты (Must have)

     Эти классы 100% нужно покрыть, потому что они содержат бизнес-логику:
     ✅ 1. WeatherCache (unit tests)
     Причины:
     это ключевой компонент
     содержит TTL
     содержит eviction strategy (максимум 10 городов)
     использует lower-case normalization

     ✅ 2. WeatherManager
     Нужен unit + integration style.

     ✅ 3. WeatherClientImpl
     Это главный класс SDK — обязательно тестировать.
     Но тут два типа тестов:
        Unit tests
        Изолируем от API → мокируем WeatherApiService.
        Integration test
        Можно написать 1–2 теста, которые реально ходят в OpenWeather API.
        Желательно отключать их аннотацией @Disabled и запускать вручную.
